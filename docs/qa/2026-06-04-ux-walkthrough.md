# UX 穿透式体验走查（规则 13 · 持续记录）

> 方法：Playwright `_electron` 驱动真实 app（构建产物）→ 每步截图 + 读 DOM → 以真实用户视角判断
> 体感 → 记录问题 → 转优化项（用户可见的优化走规则 8 样张 + 规则 7 评审）。
> harness：`tests/ux/walkthrough.mjs`（截图产物 `tests/ux/shots/`，不入库）。
> 真实外呼/花额度步骤（调 AI 生成、导出）默认不跑，先问用户。

## 本轮目的
1. 建立规则 13 的走查方法 + Playwright `_electron` harness（沉淀为长期自动化套件）。
2. 顺带对 runtime.ts 6 步拆分做**体验级回归验证**（不只是单测/build 绿）。

## 已走的腿（截图见 tests/ux/shots/）

### 01 · 项目库（landing）
- url `#/studio`（无项目时落在项目库）。渲染正常：标题「Nomi 项目库」、"30 秒体验"hero + 3 张示例卡
  （漫剧/产品 demo/短视频）、搜索框、真实项目网格（交易体系搭建/Music/Pictures/多个示例与未命名项目）。
- 体感：干净、光模式、密度合理，**无白屏无报错**。
- 端到端意义：渲染 → IPC `nomi:projects:list` → **重构后的 `projects/repository.listProjects()`** → 真实项目 → 渲染。✅

### 02 · 打开示例项目 → 工作台（studio / 生成画布）
- 点「示例：30 秒产品介绍」→ url `?step=generate#/studio?projectId=…`。
- 渲染正常：顶部导航（项目库 | 创作/生成/预览 | 素材库/模型接入/导出）、画布渲染出真实图片节点 +
  连线（一排促销图节点 + 多条 edge）、底部时间轴（图片轨/媒体轨带片段、播放头、缩放 100%）、右侧 AI 面板「Nova 生成」。
- 体感：**功能正确**，画布/素材/时间轴都真实加载；但密度高、节点类型多（文/图/视频/全景/3D），
  对纯新手偏"工程师向"（与早前 6 角色体检的发现一致）。
- 端到端意义：开项目 → IPC readProject / 列资产 → **重构后的 runtime（projects/assets/runtimePaths）** → 画布 + 资产渲染。✅

## 体验级回归结论
runtime.ts 6 步拆分后，**项目库 → 开项目 → 工作台（画布/素材/时间轴）主链路在真实 app 里渲染正常、数据正确**，
非仅 mock 单测。拆分未破坏用户可见体验。
